# 2026-05-18 lag gap特徴量と観測気温補正

> 2026-05-18の予測失敗を受け、季節固定の補正ではなく、営業日遷移と直近の気温予報バイアスに絞った保守的な改善。

言語: [English](../../en/model-improvements/model-improvement-2026-05-18-lag-gap-and-observed-weather.md) / [한국어](../../ko/model-improvements/model-improvement-2026-05-18-lag-gap-and-observed-weather.md)

---

## なぜ必要だったか

2026-05-18は、06:00-18:00の需要カーブをモデルが大きく外した。

これは単なる「暑い日補正」だけでは扱いにくい。より安全に扱える要因は二つあった。

- 前日24時間lagが異なる営業タイプから来ていた。月曜日のように前日が日曜日の場合、`lag_24h` が低需要を強く引きずることがある。
- 当日の予報気温とJMA実測気温がずれることがある。直近時間帯でその差が見えている場合、以降数時間の気温入力は需要予測前に補正する必要がある。

---

## 予測変更

LightGBM特徴量にlag gap系列を追加した。

- `lag_24h_to_last_biz_gap`
- `lag_24h_to_same_business_type_gap`
- `lag_24h_gap_x_business_hour`

これらは夏専用の補正ではない。24時間lagが、同じ営業タイプの直近需要や直近営業日の需要に比べてどれだけ低いか高いかを、モデルが直接見られるようにする。

また、気温bias補正では直近区間のJMA AMeDAS東京観測値を優先し、失敗時のみOpen-Meteo archiveへfallbackする。公式観測基準で予報が寒すぎる、または暖かすぎる場合、近い未来時間帯の気温入力を限定的に補正する。

---

## 運用変更

GitHub Actionsの手動実行で `refresh_today_forecast` を有効にすると、今日の観測済み時間帯の予測線も現在のパイプラインで再生成できる。

スケジュール実行では従来どおり過去時間帯の予測線を保持する。この手動オプションは、同じ運用日の途中でモデルや気象ソースを修正した場合に、公開済みの予測線を修正後パイプラインで描き直すためのもの。

---

## 安全性

- TEPCO予測値はモデル特徴量として使わない。
- 新しいlag gap特徴量は季節中立。
- 気温bias補正は上限を持ち、近い未来に向けて減衰する。
- スケジュール実行のデフォルトは、引き続き過去予測線の保持。

## テスト

追加・更新したテストでは以下を確認する。

- JMA AMeDAS観測気温のパース

- スケジュール実行時のデフォルト保持動作
