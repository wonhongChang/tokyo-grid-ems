# 2026-05-19 実績需要低下に基づく緩和

> 2026-05-19 の 21:00-22:00 予測線が intraday ramp guard によって過度に持ち上げられた問題への運用補正メモ。

Languages: [English](../../en/model-improvements/model-improvement-2026-05-19-observed-demand-drop-relaxation.md) / [한국어](../../ko/model-improvements/model-improvement-2026-05-19-observed-demand-drop-relaxation.md)

---

## なぜ必要だったか

20:30 の intraday スナップショットでは、19:00 の実績値を最新の基準として使っていた。

このとき双方向 ramp guard が 21:00-22:00 の速い低下を不自然な急落とみなし、予測線を上に補正した。

| 時刻 | guard 前の意図に近い値 | 公開スナップショット |
| --- | ---: | ---: |
| 21:00 | 約 28,600 MW | 30,180 MW |
| 22:00 | 約 27,500 MW | 29,580 MW |

公開された値は、従来の下落制限から直接決まっていた。

- 21:00 の下限: `31,980 - 1,800 = 30,180 MW`
- 22:00 の下限: `31,980 - 2,400 = 29,580 MW`

問題の中心は時刻そのものではない。実績需要がすでに速く下がっているなら、それが 17時でも 19時でも、近い将来の追加低下を許容すべきである。

---

## 運用変更

`ramp_guard` に実績需要低下に基づく緩和を追加した。

```yaml
intraday_correction:
  ramp_guard:
    observed_drop_relaxation:
      enabled: true
      min_recent_drop_mw: 700
      lookback_hours: 2
      skip_shape_guard: true
      max_decrease_mw_by_lead_hour: [2000, 3600, 5000]
```

直近の実績で大きな時間当たり低下が見えている場合、近い将来の低下許容幅を広げる。通常の退勤後需要低下だけでなく、特殊な早期終了パターンも、実績に現れた後は追従できる。

直近実績に意味のある低下がない場合は従来の ramp guard を維持し、説明しにくい極端な将来低下は引き続き制限する。

---

## 期待される効果

最新実績以降に需要が速く下がるだけで、将来の予測線が過度に上へ持ち上げられる現象を抑える。

TEPCO 予測に追従するルールではない。運用上の shape ルールである。

- 固定時刻より実績需要の傾きを強い根拠として扱う。
- 退勤後の業務需要は速く下がることがある。
- 特殊な早期需要低下も、実績に見えれば反映する。
- それでも説明しにくい崩壊レベルの急落は安全装置で抑える。

---

## テスト

追加テストでは以下を確認する。

- 固定時刻条件なしで、実績需要低下だけで ramp guard を緩和する。
- 実績低下が確認された場合、shape guard もスキップする。
- 極端すぎる将来低下は引き続き制限する。
- 既存の朝・午後の ramp guard 動作を維持する。
