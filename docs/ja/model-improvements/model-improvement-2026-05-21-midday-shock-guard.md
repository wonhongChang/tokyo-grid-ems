# 2026-05-21 営業日の昼時間帯ショック guard

> 12時の昼時間帯を局所的な遷移問題として扱い、1時間だけの誤差が午後の intraday 補正に波及しないようにしました。

言語: [English](../../en/model-improvements/model-improvement-2026-05-21-midday-shock-guard.md) / [한국어](../../ko/model-improvements/model-improvement-2026-05-21-midday-shock-guard.md)

---

## 背景

チャート上の12時ポイントは、12:00-13:00の需要バケットを表します。2026-05-21は、11時バケットから12時バケットにかけて実績需要が大きく低下し、その後13時バケットで一部回復しました。

これは午後全体の線形な下降トレンドではありません。intraday residual 補正が昼時間帯の dip を通常の下降モメンタムとして扱うと、13-15時の予測まで過度に下げてしまう可能性があります。運用予測モデルでは、後続の実績が確認されるまでは営業日の昼時間帯に限定された遷移として扱う方が安全です。

---

## 変更内容

TEPCO予測値はモデル入力にも guard 条件にも使用しません。新しい信号は、過去の実績需要と当日すでに観測された実績需要だけから計算します。

- `recent_same_business_type_delta_q25`: 最近の同じ営業日/非営業日タイプにおける同時間帯の実績変化量の下位25%値
- `same_day_latest_actual_hour`: 対象バケット前に得られている当日の最新実績時刻
- `same_day_latest_hourly_delta`: 当日最新実績の直前時間との差分
- `same_day_recent_hourly_delta_mean`: 当日最新の実績変化量の平均

`MiddayTransitionGuard` は、次の条件を満たす場合だけ12時の下方補正を強めます。

- 対象日が営業日
- 最近の営業日12時遷移が十分に下方向
- 当日午前の実績もすでに鈍化または低下している
- 現在の予測がその遷移 context より明確に高い

また、`IntradayResidualCorrector` は営業日12時の大きな residual を将来時間の補正平均に入れる際、低い重みで扱います。昼時間帯1バケットの誤差が午後の予測線を引き下げ続けないようにするためです。

---

## 期待される効果

モデルは次の状況をより明確に区別できます。

- 暑い営業日で需要水準自体は高い場合
- 12:00-13:00の昼時間帯だけ一時的に落ちる場合
- 後続の実績で確認される本当の午後下降トレンド

TEPCO予測に依存せず、運用中の昼時間帯の過大予測と午後への誤差波及を同時に抑えることが目的です。
