# 2026-05-15 前日比気象変化と体感温度特徴量

> 前日同時刻の需要ラグが強く効く一方で、今日の気象カーブが前日と大きく異なる場合に対応するための特徴量改善記録です。

言語: [English](../../en/model-improvements/model-improvement-2026-05-15-24h-weather-apparent-features.md) · [한국어](../../ko/model-improvements/model-improvement-2026-05-15-24h-weather-apparent-features.md)

---

## なぜ必要だったか

最近の運用予測では、`lag_24h` は有用である一方、リスクにもなり得ることが分かりました。

前日同時刻の需要が高ければ、通常はモデルがそれを参照するのは妥当です。しかし、今日の気温推移が前日より低い、高い、または最高気温の時刻が異なる場合、前日の需要は誤った基準点になることがあります。前回の改善では168時間前、つまり前週同時刻との気象レジーム差を追加しましたが、`lag_24h` に直接関係する前日比の気象変化は明示的に表現できていませんでした。

2026-05-15の予測では、この不足が見えました。モデルは前日のTEPCO実績を持っていましたが、「今日は気象面で昨日と違う」という信号をより明確に必要としていました。

## 予測改善内容

LightGBM特徴量に次の値を追加しました。

- `temp_delta_24h`: 現在同時刻の気温 - 前日同時刻の気温
- `cooling_delta_24h`: 現在同時刻の冷房degree - 前日同時刻の冷房degree
- `apparent_temp_c`: Open-Meteo体感温度
- `apparent_cooling_degree`: 体感温度から計算した冷房degree

既存の168時間気象レジーム特徴量も維持します。

- `temp_delta_168h`
- `cooling_delta_168h`

この組み合わせにより、モデルは対象時刻を前日とも前週とも比較できます。季節移行期、午前は涼しく午後だけ暖かい日、最高気温の時刻が前日より早い・遅い日で特に意味があります。

## 期待効果

`temp_delta_24h` と `cooling_delta_24h` は、モデルが `lag_24h` を信頼してよい状況と、弱めるべき状況を学ぶ助けになります。

`apparent_temp_c` と `apparent_cooling_degree` は、冷房需要を説明する2つ目の気象信号です。気温の代替ではありませんが、実際の気温だけでは体感暑さを十分に説明できない日に役立つ可能性があります。

## 設計上の境界

この変更ではTEPCO予測値をモデル入力として使いません。TEPCO予測fallbackは、確定CSV到着前の23:00実績欠損を運用上補う用途に限定して維持します。

今回の変更では09:00-15:00のような特定時間帯専用ルールは追加していません。直近の一部事象に合わせたルールより、複数の状況から学習できる汎用特徴量として残す方針です。

## 運用メモ

特徴量カラム数が30個から34個に変わったため、LightGBMモデル互換バージョンを上げました。既存の保存モデルはstale扱いとなり、次回のETLまたはintraday実行で再学習されます。
