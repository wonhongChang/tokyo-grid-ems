# 2026-05-13 日中高温ガード改善

> 2026-05-13の暖かい午後に発生した過少予測を分析し、運用モデルに追加した改善記録です。

言語: [English](../en/model-improvement-2026-05-13-daytime-heat-guard.md) · [한국어](../ko/model-improvement-2026-05-13-daytime-heat-guard.md)

---

## なぜ必要だったか

2026-05-13は、11:00-17:00 JSTの時間帯でモデル予測が実績需要を下回りました。特に日中ピーク付近に誤差が集中していました。

| 時刻 | 実績 MW | モデル MW | 誤差 MW |
|---:|---:|---:|---:|
| 12 | 33,000 | 31,751 | -1,249 |
| 13 | 33,280 | 32,014 | -1,266 |
| 14 | 33,450 | 31,479 | -1,971 |
| 15 | 33,060 | 31,652 | -1,408 |
| 16 | 32,260 | 31,167 | -1,093 |

重要なのは、単に予測が外れたことではありません。暖かい平日の午後需要が、祝日影響を含む前週ラグに引き下げられたようなパターンが見えたことです。

## 原因

モデルは同時刻の過去需要を表すラグ特徴量を使用します。

- `lag_24h`
- `lag_48h`
- `lag_168h`
- `lag_last_biz_hour`

2026-05-13の `lag_168h` は2026-05-06を参照していました。この日はゴールデンウィークの影響が残っており、通常の平日より需要が低い状態でした。

同時に、2026-05-13の午後には複数の日中時間帯で気温偏差が高く出ていました。日平均気温だけでは極端に見えなくても、午後の高温は冷房需要を押し上げる可能性があります。

要約すると次の構造です。

```text
暖かい平日午後
+ 祝日影響を含む168時間ラグ
+ 類似日補正が下方向へ動く
= 日中需要の過少予測
```

## 予測改善内容

今回の改善は保守的に設計しています。TEPCO予測値をモデル特徴量に追加したり、LightGBM予測値とTEPCO予測値をブレンドしたりしていません。

代わりに `python/forecast/adjustment.py` に2つのガードを追加しました。

1. **日中の類似日マッチング**
   - 類似日候補を選ぶ際に、10:00-17:00 JSTの日中気温偏差を使います。
   - 夜間や夕方の低い気温で、暖かい午後のシグナルが薄まることを抑えます。

2. **祝日ラグの日中ガード**
   - 同時刻168時間ラグが祝日または週末を指し、現在の日中気温偏差が高い場合に作動します。
   - 類似日補正が予測値を下方向へ押し下げることを防ぎます。
   - 必要な場合も、上方向補正は制限された範囲に留めます。

関連設定:

```yaml
adjustment:
  analogous_day:
    daytime_temp_hours: [10, 11, 12, 13, 14, 15, 16, 17]

  post_holiday_timeband_guard:
    daytime:
      min_temp_anomaly_7d: 2.0
      block_negative_shift: true
      activate_on_holiday_lag: true
      upward_offset_mw: 300
      max_upward_offset_mw: 900
```

## 異常検知改善内容

同じ調査で、p95バンドを1-50 MW程度だけ超えた場合もwarningになる問題が見つかりました。運用画面では、このような小さな超過が実際のリスクのように見えてしまいます。

そのため `python/anomaly/detector.py` では、p95超過が次のいずれかを超える場合のみwarningを作成するように変更しました。

- `warning_breach_mw: 150`
- `warning_breach_pct: 0.5`

p99超過は従来どおりcritical基準を使います。

関連設定:

```yaml
anomaly:
  spike_drop:
    warning_breach_mw: 150
    warning_breach_pct: 0.5
    critical_breach_mw: 500
    critical_breach_pct: 2.0
```

## 検証結果

2026-05-13の運用キャッシュとモデルをローカルで再実行しました。

| 指標 | 変更前 | 変更後 |
|---|---:|---:|
| 11:00-17:00 MAE | 1,110 MW | 900 MW |
| アラート数 | warning 8件 | warning 4件 |

変更後に残ったアラートは次のとおりです。

- 供給予備率に関するwarning 3件
- 継続的な残差バイアスを示すdrift warning 1件

小さなp95境界超過のspike warningは除外し、意味のある継続的な過少予測シグナルはdriftとして残しました。

全体の回帰テスト結果:

```text
230 passed
```

## 設計上の境界

今回の変更で守っている境界は次のとおりです。

- TEPCO予測値は引き続き比較対象データです。
- 23:00実績のように未確定の時間では、TEPCO予測値を一時的な実績fallbackとして使うことがあります。
- ただし、TEPCO予測値をモデル予測に直接混ぜたり、性能向上のための特徴量として使ったりはしません。

これにより、このプロジェクトはTEPCO予測を包み直したものではなく、公開データに基づく独立した予測パイプラインとして説明できます。
