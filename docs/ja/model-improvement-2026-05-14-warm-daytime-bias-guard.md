# 2026-05-14 暖かい日中の過少予測補正

> 09:00-18:00 JSTの時間帯で繰り返された暖かい日の過少予測を受けて追加した運用改善記録です。

言語: [English](../en/model-improvement-2026-05-14-warm-daytime-bias-guard.md) · [한국어](../ko/model-improvement-2026-05-14-warm-daytime-bias-guard.md)

---

## なぜ必要だったか

祝日ラグ向けの日中高温ガードを追加した後も、最近の運用データでは暖かい平日の日中に過少予測が繰り返されました。

09:00-18:00 JST基準:

| 日付 | モデルbias | モデルMAE | メモ |
|---|---:|---:|---|
| 2026-05-11 | -1,294 MW | 1,294 MW | 日中全体で過少予測 |
| 2026-05-13 | -704 MW | 751 MW | 暖かい午後がまだ低め |
| 2026-05-14 | -866 MW | 930 MW | 09:00-11:00観測時点の部分集計 |

このパターンはゴールデンウィークのラグ問題だけでは説明できません。暖かい平日そのものでも冷房需要を弱く見る傾向があり、暖候期データがさらに蓄積されるまで小さな運用ガードが必要でした。

## 予測改善内容

`python/forecast/adjustment.py` に通常の暖かい平日日中ガードを追加しました。

次の条件をすべて満たす場合のみ作動します。

- 時刻が設定された日中時間帯に入っている
- `temp_anomaly_doy >= warm_day_min_temp_anomaly_doy`
- 類似日補正がすでに予測を上方向へ動かしていない

作動すると、q50と予測バンドに小さな上方向offsetを適用します。

関連設定:

```yaml
adjustment:
  post_holiday_timeband_guard:
    daytime:
      activate_on_warm_day: true
      warm_day_min_temp_anomaly_doy: 1.0
      warm_day_upward_offset_mw: 250
```

## 設計上の境界

この補正は祝日ラグガードより小さく設計しています。

- TEPCO予測値をモデル入力として使いません。
- 固定の絶対気温ではなく、季節に対する気温偏差を基準に作動します。
- 類似日補正がすでに上方向へ動いている場合は、追加offsetを入れません。

目的はモデルを常に高めにすることではなく、暖かい日中に繰り返される過少予測を緩和することです。
