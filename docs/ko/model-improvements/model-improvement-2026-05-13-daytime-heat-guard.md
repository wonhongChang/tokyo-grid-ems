# 2026-05-13 주간 고온 보호 보정

> 2026-05-13 오후 시간대 과소예측을 분석한 뒤 추가한 운영 개선 기록입니다.

## 왜 필요했나

2026-05-13에는 11:00-17:00 JST 구간에서 모델 예측이 실제 수요보다 낮게 나왔습니다. 특히 낮 피크 주변에서 오차가 집중되었습니다.

| 시각 | 실제 MW | 모델 MW | 오차 MW |
|---:|---:|---:|---:|
| 12 | 33,000 | 31,751 | -1,249 |
| 13 | 33,280 | 32,014 | -1,266 |
| 14 | 33,450 | 31,479 | -1,971 |
| 15 | 33,060 | 31,652 | -1,408 |
| 16 | 32,260 | 31,167 | -1,093 |

핵심은 단순히 모델이 틀렸다는 점이 아니라, 따뜻한 평일 오후 수요가 휴일 영향이 남은 전주 래그값에 끌려 내려간 패턴이 보였다는 점입니다.

## 원인

모델은 같은 시간대 과거 수요를 나타내는 래그 피처를 사용합니다.

- `lag_24h`
- `lag_48h`
- `lag_168h`
- `lag_last_biz_hour`

2026-05-13의 `lag_168h`는 2026-05-06을 참조했습니다. 이 날짜는 골든위크 영향이 남아 있어 일반적인 평일보다 낮은 수요를 갖고 있었습니다.

동시에 2026-05-13 오후에는 여러 낮 시간대에서 평년 대비 높은 기온 신호가 있었습니다. 하루 평균 기온만 보면 극단적으로 보이지 않아도, 오후 고온은 냉방 수요를 밀어 올릴 수 있습니다.

요약하면 다음 구조였습니다.

```text
따뜻한 평일 오후
+ 휴일 영향이 섞인 168시간 래그
+ 유사일 보정이 아래 방향으로 움직임
= 주간 수요 과소예측
```

## 예측 개선 내용

이번 개선은 보수적으로 설계했습니다. TEPCO 예측값을 모델 피처로 넣거나 LightGBM 예측값을 TEPCO 예측과 섞지 않았습니다.

대신 `python/forecast/adjustment.py`에 두 가지 보호 장치를 추가했습니다.

1. **주간 유사일 매칭**
   - 유사일 후보를 고를 때 10:00-17:00 JST의 주간 기온 편차를 사용합니다.
   - 밤이나 저녁의 낮은 기온 때문에 따뜻한 오후 신호가 희석되는 것을 줄입니다.

2. **휴일 래그 주간 보호 보정**
   - 같은 시간대 168시간 래그가 휴일 또는 주말을 가리키고, 현재 주간 기온 편차가 높은 경우 작동합니다.
   - 이때 유사일 보정이 예측값을 아래로 밀지 못하게 막습니다.
   - 필요한 경우 제한된 범위 안에서만 위쪽 보정을 허용합니다.

관련 설정:

```yaml
adjustment:
  analogous_day:
    daytime_temp_hours: [10, 11, 12, 13, 14, 15, 16, 17]

  post_holiday_timeband_guard:
    daytime:
      min_temp_anomaly_7d: 2.0
      block_negative_shift: true
      activate_on_holiday_lag: true
      upward_offset_mw: 300
      max_upward_offset_mw: 900
```

## 이상탐지 개선 내용

같은 조사에서 p95 밴드를 1-50 MW 정도만 살짝 넘는 경우도 warning으로 잡히는 문제가 보였습니다. 운영 화면에서는 이런 작은 초과가 실제 위험처럼 보일 수 있습니다.

그래서 `python/anomaly/detector.py`는 p95 초과가 다음 기준 중 하나를 넘을 때만 warning을 만들도록 바뀌었습니다.

- `warning_breach_mw: 150`
- `warning_breach_pct: 0.5`

p99 초과는 기존처럼 critical 기준을 사용합니다.

관련 설정:

```yaml
anomaly:
  spike_drop:
    warning_breach_mw: 150
    warning_breach_pct: 0.5
    critical_breach_mw: 500
    critical_breach_pct: 2.0
```

## 검증 결과

2026-05-13 운영 캐시와 모델을 로컬에서 재실행했습니다.

| 지표 | 이전 | 이후 |
|---|---:|---:|
| 11:00-17:00 MAE | 1,110 MW | 900 MW |
| 알림 수 | warning 8개 | warning 4개 |

변경 후 남은 알림은 다음과 같습니다.

- 공급 예비율 관련 warning 3개
- 지속적인 잔차 편향을 나타내는 drift warning 1개

작은 p95 경계 초과 알림은 제거했고, 의미 있는 지속 과소예측 신호는 drift로 남겨두었습니다.

전체 회귀 테스트 결과:

```text
230 passed
```

## 설계 경계

이번 변경의 중요한 경계는 다음과 같습니다.

- TEPCO 예측값은 여전히 비교 대상 데이터입니다.
- 23:00 실측처럼 아직 확정되지 않은 시간에는 TEPCO 예측값을 임시 실측 fallback으로 사용할 수 있습니다.
- 하지만 TEPCO 예측값을 모델 예측에 직접 섞거나 성능을 높이는 피처로 사용하지는 않습니다.

따라서 이 프로젝트는 TEPCO 예측을 포장하는 구조가 아니라, 공개 데이터를 기반으로 한 독립적인 예측 파이프라인이라는 설명 가능성을 유지합니다.
