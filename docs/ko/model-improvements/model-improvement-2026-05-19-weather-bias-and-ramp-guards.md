# 2026-05-19 기온 바이어스와 당일 램프 가드

> 2026-05-19 예측 실패를 보고, 오전 램프는 살리면서 낮 시간대 과보정을 줄이기 위해 넣은 보수적인 보정 기록.

언어: [English](../../en/model-improvements/model-improvement-2026-05-19-weather-bias-and-ramp-guards.md) / [日本語](../../ja/model-improvements/model-improvement-2026-05-19-weather-bias-and-ramp-guards.md)

## 왜 필요했나

2026-05-19의 예측 실패는 한 가지 모양이 아니었다.

- 06:00-09:00은 너무 낮았다. 따뜻한 평일 아침에 수요가 빨리 올라가는 흐름을 충분히 따라가지 못했다.
- 11:00-13:00은 반대로 너무 높아졌다. 기온 바이어스 보정, warm-day 보정, intraday residual 보정이 같은 방향으로 겹쳤다.

이건 단순히 `lag_24h`가 빠진 문제가 아니었다. 전날 실측 수요는 이미 모델 입력에 들어가고 있었다. 문제는 "진짜 오전 상승"과 "오전 오차를 보고 가까운 미래를 과하게 끌어올리는 현상"을 충분히 분리하지 못한 데 있었다.

## 진단

`2026-05-19T14:45:12+09:00`에 공개된 GitHub Pages 예측을 00:00-13:00 TEPCO 실측과 비교했다.

| 시리즈 | 실측 대비 MAE | 메모 |
| --- | ---: | --- |
| 공개 모델 | 841.7 MW | 11:00-12:00 과대 예측이 큼 |
| TEPCO 예측 | 377.9 MW | 이날은 TEPCO의 시간대 모양이 더 좋았음 |

가장 큰 모델 오차는 11:00이었다. 공개 모델은 실측보다 `+2533.1 MW` 높았다. 즉, 단순히 base model이 약한 문제가 아니라 보정이 과하게 쌓인 문제도 같이 있었다.

## 예측 변경

기온 forecast bias correction을 덜 민감하게 바꿨다.

- `horizon_hours`: 4 -> 3
- `min_abs_bias_c`: 0.8 -> 1.5
- `max_abs_bias_c`: 2.5 -> 1.5
- `decay_per_hour`: 0.75 -> 0.6

이렇게 하면 기상 예보가 명확히 빗나갈 때는 보정하되, 1도 미만의 일반적인 예보 흔들림을 강한 수요 신호로 보지 않는다.

warm-day daytime guard에는 전날 같은 시간대 수요보다 과하게 높아지는 것을 막는 제한을 추가했다.

- `lag24_warm_day_cap_enabled: true`
- `lag24_warm_day_max_increase_mw: 2500`

warm-day guard가 켜진 시간대에도 예측은 `lag_24h`보다 높아질 수 있다. 다만 무제한으로 높아지지는 않는다. q50만 누르는 방식이 아니라 예측 밴드 전체를 같이 이동시켜서 밴드 폭은 유지한다.

## Intraday 변경

intraday residual correction에 가까운 미래만 보는 ramp guard를 추가했다.

10:00 이후에는 마지막 실측 수요를 기준으로 다음 1-3시간 예측 상승폭을 제한한다.

- 다음 1시간: +1200 MW
- 다음 2시간: +1500 MW
- 다음 3시간: +2000 MW

시작 시간을 10:00으로 둔 이유는 명확하다. 2026-05-19의 문제 중 하나가 06:00-09:00 오전 상승을 너무 낮게 본 것이었기 때문에, 실제 아침 램프를 가드가 눌러버리면 안 된다.

가드가 실제로 적용되면 ETL 로그에 `ramp_guard=applied`가 표시된다.

## 로컬 확인

변경 후 로컬에서 다시 생성한 예측을 같은 00:00-13:00 실측 구간과 비교했다.

| 시리즈 | 실측 대비 MAE | 메모 |
| --- | ---: | --- |
| 공개 모델 | 841.7 MW | 이번 보정 전 |
| 수정 후 로컬 모델 | 599.4 MW | 11:00 과대 예측 감소 |
| TEPCO 예측 | 377.9 MW | 이날은 아직 TEPCO가 더 좋음 |

11:00 모델 오차는 `+2533.1 MW`에서 `+659.1 MW`로 줄었다.

개선은 됐지만 완전한 승리는 아니다. 아직 06:00-09:00 오전 램프는 낮게 보고, 12:00-13:00은 높은 편이다.

## 안전 장치

- TEPCO 예측값은 여전히 모델 feature로 사용하지 않는다.
- 새 가드는 모델 출력 위에 얹는 보수적인 제한이다.
- ramp guard는 가까운 미래 몇 시간에만 적용된다.
- 10:00 이전의 실제 오전 램프는 일부러 열어둔다.
- warm-day cap은 여름 전용 규칙이 아니라, `lag_24h` 대비 과한 점프를 제한하는 목적이다.
